# Como Usar Este Repositorio

## Passo 1: Adicionar Novo Projeto

1. Copie a pasta `exemplo-projeto/`
2. Renomeie para `seu-projeto-nome/`
3. Atualize os arquivos:
   - `product-spec.md` - Especificacao do produto
   - `roadmap.md` - Roadmap e prioridades
   - `architecture.md` - Arquitetura tecnica

## Passo 2: Fluxo de Trabalho

### Com Kael (Estrategista)
1. Apresente sua ideia
2. Ele criara o `product-spec.md` completo
3. Defina problema, solucao, MVP e features

### Com Orion (Gestor)
1. Compartilhe o Product Spec
2. Ele criara o `roadmap.md`
3. Priorize features (Must/Should/Could)

### Com Nexus ou Iris (Prototipadores)
1. Recebem Product Spec
2. Criam MVP no GitHub (novo repositorio)
3. Hospeda em GitHub Pages

### Com Forge (Full-Stack)
1. Recebe MVP validado
2. Cria estrutura de producao
3. Deploy em VPS

## Passo 3: Estrutura de Arquivos por Projeto

```
seu-projeto/
+-- product-spec.md       (Criado por Kael)
+-- roadmap.md            (Criado por Orion)
+-- architecture.md       (Criado por Forge)
+-- user-stories.md       (Opcional)
+-- wireframes/           (Opcional)
+-- NOTAS.md              (Anotacoes adicionais)
```

## Dicas

1. Mantenha specs atualizadas conforme evolui
2. Use branch para features em paralelo
3. Documente decisoes importantes (ADR)
4. Valide com usuarios antes de escalar
5. Celebre pequenas vitoras!
